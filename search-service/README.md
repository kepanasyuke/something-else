# Search Service

---

## О проекте

**Search Service** — это асинхронный веб-сервис для полнотекстового поиска по документам, который сочетает удобство локального хранения (SQLite) с производительностью поискового движка (Elasticsearch).

Проект решает задачу быстрого поиска по текстовым документам с возможностью загрузки данных из CSV, просмотра результатов в веб-интерфейсе и управления документами через REST API. Он может быть полезен для прототипирования поисковых систем, создания внутренних корпоративных поисковиков или как основа для более сложных решений.

### Архитектура

Сервис построен по принципу **разделения ответственности**:

- **SQLite** — основное хранилище документов (текст, рубрики, дата). Выступает как источник истины и обеспечивает быстрое чтение по ID.
- **Elasticsearch** — поисковый индекс, оптимизированный для полнотекстового поиска на русском языке. Хранит только ID и текст документа.
- **FastAPI** — связующее звено, обрабатывает запросы, управляет загрузкой данных, синхронизирует состояния между SQLite и Elasticsearch.

Такая архитектура позволяет:
- получать высокую скорость поиска (Elasticsearch);
- сохранять все атрибуты документа в надёжном реляционном хранилище (SQLite);
- масштабировать поисковую часть независимо от хранилища.

### Принципы разработки

- **Асинхронность** — все операции с БД и Elasticsearch выполняются асинхронно, что обеспечивает высокую пропускную способность.
- **Чистота кода** — код разделён на короткие функции (не более 30–50 строк), каждая функция имеет одну точку выхода, запрещены `break` и `continue` вместе с `return`.
- **Тестируемость** — покрытие составляет 87%, все ключевые сценарии проверены.
- **Простота развёртывания** — один Docker-команда запускает весь стек, переменные окружения управляются через `.env`.

### Как это работает

1. **Загрузка данных**: CSV-файл с колонками `text`, `created_date`, `rubrics` автоматически загружается при первом запуске (если база пуста). Для каждого документа формируется SQLite-запись и отдельно — индекс в Elasticsearch.
2. **Поиск**: пользователь вводит запрос в интерфейсе или через API. Elasticsearch возвращает ID документов, по которым SQLite отдаёт полные записи. Результаты сортируются по дате (сначала новые).
3. **Удаление**: документ удаляется одновременно из SQLite и Elasticsearch (если одна операция не удалась, сервис возвращает частичный успех).
4. **Интерфейс**: веб-страница поддерживает подсветку найденных слов, пагинацию и экспорт в Excel.

### Особенности

- **Автоиндексация** — при запуске сервис проверяет, есть ли данные в SQLite; если нет — загружает CSV.
- **Поддержка русского языка** — анализатор `russian` в Elasticsearch обеспечивает корректную стемминг.
- **Готовность к продакшену** — предусмотрены обработка ошибок, логирование, таймауты, повторные попытки.

## Возможности

- **Загрузка данных** из CSV (локально или по URL).
- **Индексация** в Elasticsearch с поддержкой русского языка.
- **Полнотекстовый поиск** с пагинацией (постраничная загрузка).
- **Веб-интерфейс**: поиск, просмотр, удаление документов, экспорт в XLSX.
- **Подсветка** найденных ключевых слов.
- **REST API** с документацией OpenAPI (`/docs`).
- **Асинхронная работа** с SQLite и Elasticsearch.
- **Код**: принцип «один выход на функцию», нет `break/continue`, функции разбиты на короткие (до 30–50 строк).
- **87% покрытие** тестами (pytest + coverage).

## Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone <repository-url>
cd search-service
```

### 2. Создайте виртуальное окружение и установите зависимости

```bash
make install
```

Или вручную:

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настройте окружение

```bash
make env
```

Эта команда скопирует `.env.example` в `.env`. При необходимости отредактируйте параметры (см. раздел «Конфигурация»).

### 4. Запустите Elasticsearch

Локально через Docker:

```bash
docker run -d -p 9200:9200 -e "discovery.type=single-node" elasticsearch:8.11.0
```

Или используйте `docker-compose up -d` (если есть `docker-compose.yml`).

### 5. Загрузите данные

Автоматически при первом запуске, если есть `posts.csv` в папке `data/`.

### 6. Запустите сервер

```bash
make run
```

Сервис будет доступен по адресу: `http://localhost:8000`

## Использование

### Веб-интерфейс

Откройте `http://localhost:8000`, введите запрос в поле поиска.

## Конфигурация (.env) 

| Параметр | Описание |
|----------|----------|
| `DATABASE_URL` | Путь к SQLite (по умолчанию `sqlite:///data/documents.db`) |
| `ES_HOST` | Адрес Elasticsearch (по умолчанию `localhost:9200`) |
| `SEARCH_SIZE` | Количество результатов поиска (по умолчанию 30) |
| `BATCH_SIZE` | Размер пачки при загрузке данных (по умолчанию 500) |
| `LOG_LEVEL` | Уровень логирования (`INFO`, `DEBUG` и т.д.) |
| `SQLITE_SYNCHRONOUS` | Режим синхронности SQLite (для продакшена рекомендуется `NORMAL`) |
| `CSV_PATH` | Путь к CSV-файлу с данными (по умолчанию `data/posts.csv`) |
